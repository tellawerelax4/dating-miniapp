# FIGMA IMPLEMENTATION GUIDE

Version: 1.0

---

# PURPOSE

Defines how the Figma project must be generated.

---

# FILE STRUCTURE

Pages:

- Cover
- Design System
- Components
- Mobile Screens
- Admin Panel
- Prototype

---

# DESIGN SYSTEM

Create:

Color Variables

Typography Styles

Spacing Tokens

Radius Tokens

Shadow Tokens

Icon Library

Component Variants

---

# COMPONENTS

Create reusable components for:

Buttons

Inputs

Bottom Navigation

Profile Card

Interest Chip

Dialogs

Bottom Sheets

Avatar

Toast

Skeleton

Empty State

Loading State

---

# AUTO LAYOUT

Every screen must use Auto Layout.

No absolute positioning unless required.

---

# VARIABLES

Use Variables for:

Colors

Spacing

Radius

Typography

Themes

---

# THEMES

Generate:

Light Theme

Dark Theme

Every component must support both themes.

---

# PROTOTYPE

Connect every screen.

Include:

Registration

Discovery

Likes

Matches

Profile

Settings

Admin Panel

Dialogs

Bottom Sheets

---

# RESPONSIVE

Support:

320 px

360 px

390 px

430 px

Layouts must scale correctly.

---

# NAMING

Use consistent names.

Example:

Button/Primary

Button/Secondary

Card/Profile

Sheet/Report

Dialog/Delete

Screen/Discover

Screen/Profile

---

# HANDOFF

Every component must be Developer Ready.

Use:

Auto Layout

Variants

Variables

Constraints

No detached components.

---

# FINAL RESULT

The Figma project must be fully organized, editable, reusable and ready for direct implementation without redesign.
